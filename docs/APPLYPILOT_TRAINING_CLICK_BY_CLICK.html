ApplyPilot Training: Click-by-Click

Use this when onboarding new users. Follow each step exactly in dashboard mode.

Start dashboard first:
applypilot dashboard-serve --multi-user or docker compose up --build

Step 1: Open Dashboard and Understand Main Areas

  1. Open http://127.0.0.1:8765.
  2. Locate three key areas: job filters, pipeline controls, setup panel.
  3. Use this screen as the main operating center.

Step 2: Use Pipeline Controls Correctly

  1. Run stages in order: Discover -> Enrich -> Score.
  2. Use toggles like Selected only when targeting specific jobs.
  3. Use Stop if a run is taking too long.

Step 3: Manage Jobs from Card Actions

  1. Click Pick on high-fit jobs you want to tailor/apply first.
  2. Use Applied, Failed, Block as needed.
  3. Avoid deleting jobs unless they are clearly irrelevant.

Step 4: Complete Full Profile Editor (Required)

  1. Open Setup -> Full Profile Editor.
  2. Fill all critical fields: name/email, work authorization, compensation, experience.
  3. Click Save full profile.

Step 5: Configure Tailoring Intelligence

  1. Keep Role Pack on Auto unless you are role-specific.
  2. Use Resume Template Builder to add skills, certifications, education, metrics.
  3. Click Save tailoring config after changes.

Step 6: Build Search Config

  1. Use builder mode for non-technical users.
  2. Set countries/locations/roles/results thoughtfully.
  3. Save searches.yaml and run Discover again.

Step 7: Tailor Only Selected Jobs (Best Practice)

  1. Pick jobs first.
  2. Tick Selected only in pipeline controls.
  3. Run Tailor or click Regenerate tailored resumes now.

This prevents wasting tokens on low-priority jobs and improves output quality.

Step 8: Final QA Before Applying

  1. Open tailored resume and cover letter artifacts for top jobs.
  2. Check factual accuracy (companies, dates, metrics).
  3. Run apply stage (dry-run first if needed).

Trainer Checklist

  • User has unique account (multi-user mode) and isolated workspace.
  • User has GEMINI_API_KEY in their own workspace .env.
  • Profile, resume text, and searches are saved successfully.
  • User can run discover/enrich/score and tailor selected jobs.
  • User understands Applied/Failed/Block lifecycle actions.

Training complete.
