1) What ApplyPilot Does

ApplyPilot is an autonomous job pipeline. It discovers jobs, enriches descriptions, scores fit, tailors your resume, drafts cover letters, and can automate browser-based applications.

StagePurposeMain Output
DiscoverFind jobs from configured sourcesRows in applypilot.db
EnrichFetch complete job descriptionsfull_description populated
ScoreAssign fit score vs your profilefit_score, reasoning
TailorCreate per-job resume contenttailored_resumes/*
CoverGenerate per-job cover letterscover_letters/*
PDFProduce polished PDFsPDF artifacts per job
ApplyBrowser submission automationApplied/failed status updates

2) Architecture and Data Model

Workspace basics

  • profile.json: your profile data (personal, work auth, experience, skills boundaries, resume facts, tailoring controls, EEO defaults).
  • searches.yaml: query, locations, boards, exclusions, defaults.
  • resume.txt: source resume text used by scorer/tailor.
  • .env: secrets and runtime settings (GEMINI_API_KEY, etc.).
  • applypilot.db: jobs, scoring, pipeline state, status.
  • tailored_resumes/, cover_letters/: generated artifacts.

Workspace location resolution

  1. If APPLYPILOT_DIR is set, that path is used.
  2. Otherwise the app auto-picks between ~/.applypilot and ~/.applypilot-data based on existing data completeness.

Tier gating

  • Tier 1 (Discovery): no LLM key required.
  • Tier 2 (AI scoring/tailoring): requires GEMINI_API_KEY (or OpenAI/local provider config).
  • Tier 3 (Auto-apply): requires Claude CLI + Chrome.

4) First-Time Setup

Minimum required to tailor

  1. Add GEMINI_API_KEY to workspace .env.
  2. Save profile data (quick profile or full profile editor).
  3. Paste/upload resume content (at least resume.txt).
  4. Configure searches.yaml.
  5. Run discover -> enrich -> score -> tailor.
Docker multi-user note: each user has their own workspace. User secrets go in
docker-data/multi/workspaces/<username>/.env.

5) Dashboard Guide (End-to-End)

A) Top controls and filtering

  • Score quick filters: 7+, 8+, 9+.
  • Text filters for role/site/status.
  • Bulk delete shown or role subset.

B) Job cards

  • Pick/Unpick: mark jobs as selected for targeted pipeline runs.
  • Applied / Failed / Block / Delete actions.
  • Copy helper CLI commands per job.
  • Expandable full job description panel.

C) Pipeline Controls

  • One-click stage groups: All, Prep-only, Apply-only.
  • Single stages: discover, enrich, score, tailor, cover, pdf.
  • Runtime toggles: stream, dry-run, selected-only, skip discovery engines.
  • Stop and clear run controls.
  • Pipeline Console: recent runs, logs, status polling.

D) Setup panel

Diagnostics

  • Health ping, Setup API, Pipeline API checks.
  • Copy debug payload for support/debugging.

Quick Profile card

  • Fast fields: name/email/phone/city/country/linkedin/target role/years.
  • Use for quick edits only.

Full Profile Editor card

  • Comprehensive structured editing for all major profile sections.
  • Includes dropdowns and yes/no checkboxes for consistency.
  • Advanced JSON area for direct patch editing.
  • Buttons: Apply form to JSON, Apply JSON to form, Sync from loaded profile, Save full profile.

Resume card

  • Save resume.txt (primary input for scoring/tailoring).
  • Upload resume.pdf (optional reference).

Tailoring Intelligence card

  • Role pack: auto/data_bi/engineering/support.
  • Draft candidates: 2-3.
  • Resume Template Builder (non-JSON editing for skills/facts/sections).
  • Raw JSON editors for skills boundary, facts, sections, validation, synonyms.
  • Regenerate tailored resumes now: persists settings and runs tailor stage.

Job Search Config card

  • Builder mode: country/locations/roles/boards/smart sites/exclusions.
  • Advanced YAML mode for full direct editing of searches.yaml.
  • Validation guardrails for board names and numeric limits.

7) Tailoring and Resume Generation Logic

How tailoring decides content

  1. Reads each job description and extracts required terms/signals.
  2. Uses your profile boundaries/facts to keep claims grounded.
  3. Rewrites summary + bullets + projects per job context.
  4. Outputs per-job artifacts, not one generic resume.

Current hardening behavior

  • Filename prefers human name over username where possible.
  • Projects section gets fallback content if model output is empty.
  • Education section preserves key school/education-level context when sparse.
  • Resume sections accept list and legacy newline-string inputs.

Selective tailoring (important)

  1. Pick desired jobs on cards.
  2. Enable Selected only in Pipeline Controls.
  3. Run Tailor stage (or Regenerate tailored resumes now).

8) Pipeline Controls and Safe Execution

Recommended default

  • Discover -> Enrich -> Score first (validate quality and fit).
  • Then tailor selected high-fit jobs only.
  • Use dry-run for apply stage before full submission.

Stage combos

# Dashboard equivalents (CLI)
applypilot run discover enrich score
applypilot run tailor cover pdf
applypilot run all

10) Multi-User Operations

What isolation means

  • Each account gets an isolated workspace folder and DB artifacts.
  • User A profile/searches/resume do not overwrite User B data.
  • In Docker multi-user mode, per-user roots are under docker-data/multi/workspaces/<username>.

Important operational rule

API keys and site passwords are per-user in .env. If tailoring says key missing, check the logged-in user workspace.

11) Workspace Transfer / Backup / Restore

Backup current workspace

applypilot workspace-export --out ~/applypilot-transfer.zip --include-env --include-generated
# clean export (no db, no generated):
applypilot workspace-export --out ~/applypilot-transfer-clean.zip --no-db --no-generated --include-env

Restore on another machine

applypilot workspace-import ~/applypilot-transfer.zip --overwrite

Docker transfer commands

docker compose run --rm applypilot applypilot workspace-import /transfer/applypilot-transfer-clean.zip --overwrite
docker compose run --rm applypilot applypilot workspace-export --out /transfer/applypilot-transfer.zip --include-env --include-generated

13) Recommended Daily Workflows

Workflow A: High-volume search + selective tailoring

  1. Run discover/enrich/score on broad queries.
  2. Filter by score 7+ and role keywords.
  3. Pick only best-matching jobs.
  4. Run tailor on selected only.
  5. Review outputs, then apply.

Workflow B: Daily maintenance

  1. Refresh searches and run discover.
  2. Block noise domains/jobs aggressively.
  3. Keep profile facts and metrics up to date.
  4. Archive backups weekly with workspace-export.

15) Security and Data Hygiene

  • Never commit real .env keys or transfer zips to git.
  • Use workspace-export --no-db --no-generated for clean handoff.
  • Only share archives with --include-env when absolutely necessary.
  • Use multi-user mode for shared deployments to isolate users safely.
  • Regularly review blocked sites and archived/skipped jobs.
